1. Set `"type": "module"` at the top level (required for ES module `export`/`import` syntax in lib/bidi-detect.js and content.js)

2. Add the `"scripts"` entry:
```json
"scripts": {

}
```

3. Add the `"jest"` configuration block:
```json
"jest": {
  "testEnvironment": "node",
  "transform": {},
  "testMatch": ["**/tests/**/*.test.js"]
}
```

Note: `"transform": {}` disables Babel transform so Jest uses native ESM. `NODE_OPTIONS=--experimental-vm-modules` enables Jest 29's native ESM support. The default testEnvironment is "node" — individual test files that need DOM APIs will override with `@jest-environment jsdom` docblock.

Do NOT install `@babel/core`, `babel-jest`, or `@babel/preset-env` — the project uses zero-transform native ESM, not Babel.
