## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between checkbox and selector text; gap between row buttons |
| sm | 8px | Internal padding for selector rows; gap between header elements |
| md | 12px | Popup body padding (all four sides) |
| lg | 16px | Section separation between header, selector list, and footer |
| xl | 24px | Confirm dialog internal padding |

Exceptions:
- Popup body width: 280px (fixed — locked in CONTEXT.md; set on `body { width: 280px }`)
- Touch/click targets for buttons: minimum 28px height (Delete ✕ button, checkbox)
- Confirm dialog buttons: minimum 32px height for clear tap affordance
- **`md: 12px` justification:** The popup canvas is 280px wide. Using 16px body padding would leave only 248px of usable content width, making truncated selector text harder to read. 8px would feel cramped against the border. 12px is the deliberate midpoint that preserves readability within the constrained popup width. It is a multiple of 4 and is consciously preferred over both 8px and 16px for this specific layout.

Source: CONTEXT.md (width: ~280px), defaults for all other values.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Domain name (header) | 14px | 600 (semibold) | 1.2 |
| Selector text (row) | 13px | 400 (regular) | 1.4 |
| Button labels / menu items | 13px | 400 (regular) | 1.2 |
| Confirm dialog body text | 13px | 400 (regular) | 1.5 |

Font stack for all text: `system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`

Selector text overflow: `overflow: hidden; text-overflow: ellipsis; white-space: nowrap`
Full selector text exposed via `title` attribute (native tooltip on hover).

**Note on 13px/14px proximity:** The 1px size difference between the domain name (14px) and selector text (13px) is intentional. Hierarchy is not established by size alone — it is reinforced by weight (600 semibold vs. 400 regular) and vertical position (header above list). In the 280px popup canvas, a larger size jump (e.g., 16px vs. 13px) would create disproportionate visual weight given the compact height of each row.

Source: defaults informed by Chrome extension UI conventions; selector truncation from CONTEXT.md.

|------|-------|-------|
| Dominant (60%) | #FFFFFF | Popup background, selector list background |
| Secondary (30%) | #F3F4F6 | Selector rows (zebra — alternating, or hover state), confirm dialog background, actions dropdown background |
| Accent (10%) | #2563EB | Master toggle ON state fill, hover highlight outline on page elements |

Accent (#2563EB) reserved for:

## Primary Visual Anchor

The master toggle in the header (`<input type="checkbox" id="master-toggle">` styled as a pill toggle) is the primary visual anchor of the popup. It is the only element rendered at weight 600 semibold context (the domain name beside it is also 600, forming a single unified header unit). The toggle controls all behavior below it — when OFF, the entire selector list dims to opacity 0.4 and becomes non-interactive. Executor must ensure the toggle is visually prominent and is the first focusable element in tab order.

## Interaction States

### Master Toggle (POP-01)
- **ON**: toggle visually checked; selector list fully opaque, pointer-events: auto
- **OFF**: toggle visually unchecked; selector list has `opacity: 0.4; pointer-events: none`
- **Write on change**: immediate — no save button; calls `setDomainConfig(hostname, config)`

### Selector Rows (POP-02)
- **Default**: full row visible at all times — no hover-reveal controls
- **Hover**: row background shifts to secondary color (#F3F4F6); triggers HIGHLIGHT_SELECTOR message to content.js
- **Mouse leave**: row background returns to white; triggers CLEAR_HIGHLIGHT message to content.js
- **Checkbox change**: immediate storage write, no optimistic UI — wait for `setDomainConfig` to resolve
- **Delete (✕) click**: immediate removal — no confirmation; `config.selectors.splice(index, 1)` then `setDomainConfig`

### Add Selector Button (POP-03)
- **Click**: shows inline message element `#add-selector-msg` with text "Coming soon — element picker in next update"
- Message appears below the selector list; disappears if user clicks elsewhere or after 3 seconds (auto-hide)
- No modal, no row creation, no navigation

### Actions Dropdown (POP-05)
- **Button label**: `Actions ▾` (Unicode ▾ U+25BE)
- **Click**: toggles `#actions-menu` visibility (hidden ↔ visible)
- **Outside click**: closes the menu (document-level click listener with `event.target.closest` guard)
- **Keyboard**: Escape key closes the dropdown if open

Actions menu items and their behaviors:

| Item | Label | Behavior |
|------|-------|----------|
| Export Config | "Export Config" | Calls `getAllConfigs()`, creates JSON Blob, triggers download as `hebrew-rtl-config-YYYY-MM-DD.json` |
| Delete All | "Delete All Selectors" | Shows confirm dialog (see below) |
| Shortcuts | "Keyboard Shortcuts" | Calls `chrome.tabs.create({ url: 'chrome://extensions/shortcuts' })` |

Note: "User Guide" and "Keyboard Shortcuts" are merged into a single entry ("Keyboard Shortcuts") — both point to the same URL; having two is confusing. Source: RESEARCH.md open question #2 resolution (Claude's discretion).

### Confirm Dialog — Delete All (POP-05)
- **Trigger**: clicking "Delete All Selectors" in the Actions menu
- **Implementation**: custom HTML `<div id="confirm-dialog">` within popup DOM; `window.confirm()` is blocked in Chrome extension popups
- **Dialog text**: `"Delete all selectors for [domain]? This cannot be undone."`
  - `[domain]` is the actual hostname (e.g. `chatgpt.com`)
- **Buttons**: "Confirm" (destructive — red-600 border or text) and "Cancel" (neutral)
- **Confirm action**: sets `config.selectors = []`, calls `setDomainConfig(hostname, config)`, hides dialog, re-renders selector list in empty state
- **Cancel action**: hides dialog, no storage change

### Not Available State
- **Trigger**: popup opened on a `chrome://` page, PDF, or extension page where `tab.url` is inaccessible
- **Display**: domain name area shows "Not available on this page" (muted text); all controls hidden

### Empty State (no selectors configured)
- **Selector list area**: shows prompt text "No selectors configured — add one with +" in muted text (#6B7280)
- **Add Selector button**: remains visible and enabled
- **After Delete All confirmed**: popup re-renders to this empty state

## Layout Structure

Popup fixed width: 280px. Height: auto (scrollable selector list if needed).

```
┌─────────────────────────────────┐  ← 280px wide
│  chatgpt.com          [toggle]  │  ← header (12px padding, 14px semibold domain name)
├─────────────────────────────────┤
│  .message-box  [✓] [✕]          │  ← selector row (13px text, truncated)
│  .reply-box    [✓] [✕]          │
│  ...                            │
│  (scrollable if > 5-6 rows)     │
├─────────────────────────────────┤
│  [+ Add Selector]  [Actions ▾]  │  ← footer
└─────────────────────────────────┘

When Actions ▾ is open (dropdown below the button, inside popup bounds):
┌─────────────────────────────┐
│  Export Config              │
│  Delete All Selectors       │
│  Keyboard Shortcuts         │
└─────────────────────────────┘

Confirm dialog overlays the popup body:
┌─────────────────────────────────┐
│  Delete all selectors for       │
│  chatgpt.com? This cannot       │
│  be undone.                     │
│                                 │
│         [Cancel]  [Confirm]     │
└─────────────────────────────────┘
```

Selector list max-height: 240px with `overflow-y: auto` — prevents popup from growing too tall on domains with many selectors.

## Component Inventory

All components are hand-built HTML elements — no component library.

| Component | HTML element | Notes |
|-----------|-------------|-------|
| Master toggle | `<input type="checkbox" id="master-toggle">` styled as a pill toggle | CSS-only: appearance:none + ::before pseudo-element for thumb |
| Selector row | `<div class="selector-row">` | Contains: `<span class="selector-text">`, `<input type="checkbox">`, `<button class="delete-btn">` |
| Add Selector button | `<button id="add-selector-btn">` | Full-width or left-aligned in footer |
| Actions button | `<button id="actions-btn">` | Toggles `#actions-menu` |
| Actions menu | `<ul id="actions-menu" hidden>` with `<li><button>` items | `hidden` attribute; positioned absolute below button |
| Confirm dialog | `<div id="confirm-dialog" hidden>` | Shows/hides inline; not a real modal (no backdrop) |
| Add Selector message | `<p id="add-selector-msg" hidden>` | Auto-hidden after 3 seconds |
| Empty state | `<p id="empty-state">` | Shown when `selectors.length === 0` |
| Not available state | `<p id="not-available">` | Shown when hostname resolution fails |

**Delete button accessibility:** The delete button (`<button class="delete-btn">`) must carry both `title="Remove selector"` (native tooltip) and `aria-label="Remove selector"` (screen reader label). The visible content is `✕` (Unicode U+2715) which is not reliably announced by all screen readers, so the explicit `aria-label` is required.
