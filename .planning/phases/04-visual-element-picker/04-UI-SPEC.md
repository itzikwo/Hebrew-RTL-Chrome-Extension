# Phase 4 — UI Design Contract

> Visual and interaction contract for Phase 4: Visual Element Picker.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — vanilla CSS only |
| Icon library | Unicode symbols inline (× for close, ▾ for dropdown) |
| Font | system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif |

**Source:** existing popup.html inline styles + 04-CONTEXT.md (zero-build, no framework, no bundler).

Phase 4 introduces two new injected surfaces — the **picker banner** and the **Selector Builder overlay** — both isolated via Shadow DOM (open mode). Their internal CSS must be self-contained and must not inherit from or leak into the host page.

Exceptions:
- Picker banner height: 40px (fixed — fits a single line of text + 8px vertical padding each side)

**Source:** popup.html (padding: 8px body, gap: 8px header, margin-bottom: 16px, padding: 24px confirm-dialog).

## Typography

Four roles used across both picker surfaces and overlay. All sizes match the existing popup type scale:

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 13px | 400 | 1.4 | Ancestor dropdown entries, filter labels, selector preview string, banner text |
| Label | 14px | 400 | 1.2 | Tooltip tag/class display, overlay section labels |
| Heading | 14px | 600 | 1.2 | Overlay panel title ("Configure Selector"), domain name (matches popup header) |
| Mono | 13px | 400 | 1.4 | Computed selector preview string (font-family: monospace) |

Weights declared: 400 (regular) and 600 (semibold) — no other weights permitted.

**Source:** popup.html (font-size 13px body/labels, 14px domain-name at weight 600, line-height 1.2 and 1.4 established).

## Color

All colors extend the existing popup palette. No new hues introduced except the picker accent (orange/amber), which is locked in CONTEXT.md to be distinct from the existing blue `#2563EB` highlight.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FFFFFF | Page overlay panel background, banner background |
| Secondary (30%) | #F3F4F6 | Overlay panel section backgrounds, dropdown background, filter row backgrounds |
| Accent — interactive blue (10%) | #2563EB | Save button background, active checkbox in overlay (matches Phase 3 POP-04 highlight color) |
| Accent — picker orange | #F59E0B | Picker hover outline (2px solid), banner left border accent stripe |
| Destructive | #DC2626 | Cancel button text (matches existing delete-btn and confirm-ok pattern) |

Accent reserved for:
- `#2563EB` — Save button background in Selector Builder; active state of master toggle (inherited from popup); matched-element highlight via POP-04 live preview
- `#F59E0B` — picker hover outline on the targeted element; 4px left border stripe on the picker banner to signal active picker mode

Additional semantic colors (inherited from popup.html, not new):
- Border / divider: `#E5E7EB`
- Text primary: `#111827`
- Text secondary / muted: `#6B7280`
- Destructive hover background: `#FEE2E2`

**Source:** popup.html color values + 04-CONTEXT.md decision ("Use orange/amber e.g. #F59E0B distinct from #2563EB").

## Component Inventory

### 1. Picker Banner (injected into host page via Shadow DOM, open mode)

**Purpose:** Signals that picker mode is active. Stays visible while user hovers to pick an element.

**Anatomy:**
- Fixed position, top: 0, left: 0, right: 0, z-index: 2147483647 (max safe z-index to clear all host page stacking contexts)
- Height: 40px
- Background: #FFFFFF
- Bottom border: 1px solid #E5E7EB
- Left border: 4px solid #F59E0B (orange accent stripe signals picker active)
- Content: single line of text, centered vertically, padding 0 16px

**Text content:**
"Click an element to configure Hebrew RTL — Esc to cancel"
- Size: 13px, weight 400, color #111827, line-height 1.4
- Source: exact wording from 04-CONTEXT.md `<specifics>`

**States:**
- Visible: displayed while picker is active (hover phase)
- Hidden: removed on Escape, on element click-to-select, or on Cancel in Selector Builder

---

### 2. Picker Hover Tooltip (injected into host page, Shadow DOM)

**Purpose:** Shows element identity when hovering during picker mode.

**Anatomy:**
- Position: absolute, offset 8px above or below the hovered element (flip if near viewport top)
- Background: #111827 (dark, high contrast on any page)
- Color: #FFFFFF
- Border-radius: 4px
- Padding: 4px 8px (xs sm)
- Font-size: 13px, weight 400, line-height 1.4
- Max-width: 320px, overflow: hidden, text-overflow: ellipsis, white-space: nowrap
- z-index: 2147483646 (one below banner)

**Content format:** `tag.class1.class2` or `tag#id` then newline then `selector.string`
- Example: `div.message-content` / `div.message-content`
- Closed shadow root case: "Cannot pick inside this element" (italic, color #6B7280)

**Source:** 04-CONTEXT.md decisions (tooltip shows tag, classes, computed selector; closed shadow root copy).

---

### 3. Picker Hover Outline (applied to host page element, NOT via Shadow DOM)

**Purpose:** Visually marks the element currently under the cursor.

**Implementation:** inline style set on the hovered element directly:
```
outline: 2px solid #F59E0B;
outline-offset: 2px;
```
Removed on mouseleave or picker deactivation. Previous outline value is saved and restored on cleanup.

**Source:** 04-CONTEXT.md ("picker hover outline uses orange/amber #F59E0B, 2px").

---

### 4. Selector Builder Overlay (fixed panel, Shadow DOM, open mode)

**Purpose:** Post-click UI where user refines the selector before saving.

**Anatomy:**
- Position: fixed, bottom: 0, left: 0, right: 0
- Background: #FFFFFF
- Top border: 1px solid #E5E7EB
- Box-shadow: 0 -2px 12px rgba(0, 0, 0, 0.12) (lifts panel above page content visually)
- z-index: 2147483647
- Padding: 16px 24px (md lg)
- Max-height: 320px
- Overflow-y: auto

**Internal layout (top to bottom):**

1. Panel title row
   - Left: "Configure Selector" — 14px, weight 600, color #111827
   - Right: close/cancel button "×" — 20px, color #6B7280, background none, border none, cursor pointer, min 32px tap target

2. Ancestor dropdown row
   - Label: "Element level" — 13px, weight 400, color #6B7280
   - `<select>` element — full width, 13px, weight 400, color #111827, background #F3F4F6, border 1px solid #E5E7EB, border-radius 4px, padding 6px 8px, min-height 32px
   - Each `<option>`: `tag.class1.class2` or `tag#id` format
   - Up to 10 entries (clicked element + up to 9 ancestors, stops at `<body>`)

3. Class filters section (shown only when ancestor has classes)
   - Label: "Class filters (optional)" — 13px, weight 400, color #6B7280
   - Checkbox list: one row per class on the selected ancestor element
   - Each row: 16px checkbox + 8px gap + class name in 13px monospace, weight 400, color #111827
   - Row padding: 4px 0 (xs)

4. Attribute filters section (shown only when ancestor has data-* or role attributes)
   - Label: "Attribute filters (optional)" — 13px, weight 400, color #6B7280
   - Checkbox list: one row per relevant attribute (data-*, role, aria-*)
   - Each row: 16px checkbox + 8px gap + `[attr="value"]` in 13px monospace, weight 400, color #111827

5. Selector preview row
   - Label: "Selector" — 13px, weight 400, color #6B7280
   - Preview string — 13px monospace, weight 400, color #111827, background #F3F4F6, padding 6px 8px, border-radius 4px, word-break: break-all

6. Action buttons row
   - Aligned to right (justify-content: flex-end), gap 8px
   - Cancel button: 13px, weight 400, color #DC2626, background none, border 1px solid #DC2626, border-radius 4px, padding 6px 16px, min-height 32px
   - Cancel hover: background #FEE2E2
   - Save button: 13px, weight 400, color #FFFFFF, background #2563EB, border none, border-radius 4px, padding 6px 16px, min-height 32px
   - Save hover: background #1D4ED8 (one Tailwind step darker, stays in blue family)

**Source:** 04-CONTEXT.md decisions (fixed bottom panel, ancestor dropdown, class+attribute filters, live preview, Save/Cancel).

## Interaction Contract

### Picker State Machine

| State | Cursor | Banner | Hover Outline | Tooltip | Overlay |
|-------|--------|--------|---------------|---------|---------|
| Inactive | default | hidden | none | none | hidden |
| Active — hovering | crosshair | visible | #F59E0B outline on hovered element | visible | hidden |
| Active — element clicked | crosshair | visible | outline stays on selected element | hidden | visible |
| Saving | crosshair | visible | outline stays on selected element | hidden | visible (disabled) |
| Cleanup (cancel or save complete) | default | hidden | removed | none | hidden |

### Event Interception (picker active)

- All `click` events on the page intercepted at document level (capture phase, `stopPropagation` + `preventDefault`) except clicks within the picker's own Shadow DOM overlay
- `mousemove` drives hover outline and tooltip updates — debounced at 16ms (one animation frame)
- `keydown` on Escape triggers full reset from any picker state

### Live Preview Highlight

- When the ancestor dropdown selection changes: send existing `highlight` message (same as POP-04 hover highlight) using the current generated selector
- Uses the existing `#2563EB` blue highlight — same as popup selector-row hover, so user recognizes the pattern
- Updating a class/attribute filter checkbox also re-triggers the live highlight immediately (no debounce — checkbox interaction is explicit)

### Shadow DOM Isolation

- Picker banner and Selector Builder overlay are attached to a `div` appended to `document.body` using `element.attachShadow({ mode: 'open' })`
- All styles are injected as a `<style>` tag inside the shadow root — no styles inherit from the host page
- Font fallback: shadow root inherits `font-family` from host page body via CSS inheritance (acceptable — system font stack is universal)

### Click Suppression

- Active during hover phase only (before element is clicked)
- Removed once an element is clicked and the Selector Builder overlay opens (page is already frozen in place at that point)

## Accessibility Contract

- Picker banner: `role="status"` + `aria-live="polite"` so screen readers announce picker activation
- Overlay panel: `role="dialog"` + `aria-modal="true"` + `aria-label="Configure Selector"`
- Focus management: when Selector Builder overlay opens, focus moves to the ancestor `<select>` element; when overlay closes, focus returns to the document body (popup is already closed)
- All interactive controls in overlay: minimum 32px touch/click target height
- Escape key exits picker from any state (banner or overlay open)
- Color contrast: text #111827 on #FFFFFF — 16.1:1 (passes AAA); text #6B7280 on #FFFFFF — 4.6:1 (passes AA); white text on #2563EB Save button — 4.7:1 (passes AA)
