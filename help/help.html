Hebrew RTL — Help

v0.2.0

What this extension does

Hebrew RTL automatically fixes the text direction of Hebrew content on any website. It works at the individual element level — only the elements containing Hebrew text are changed, leaving the rest of the page untouched.

It detects Hebrew using Unicode analysis: if the first strong character is Hebrew, or if 30% or more of a text's letters are Hebrew, the element gets direction: rtl and text-align: right.

Turning RTL correction on or off

Master toggle — The toggle switch at the top of the popup enables or disables all RTL corrections for the current website. When it is off, the selector list dims and no changes are applied.

Per-selector toggle — Each row in the selector list has its own checkbox. Uncheck a row to disable that specific selector while keeping others active.

Keyboard shortcut — Press Ctrl+Shift+H (or ⌃+Shift+H on Mac) to toggle the master switch for the current domain without opening the popup.

To change the keyboard shortcut, open the Actions menu → Keyboard Shortcuts. This opens Chrome's built-in extension shortcuts page.

Adding a selector for a new site

Use the visual element picker to target specific elements on any page:

  1. Navigate to the website you want to fix.
  2. Click the extension icon to open the popup, then click + Add Selector. The popup closes and picker mode activates.
  3. Hover over the page — elements highlight in amber as your cursor moves over them. A tooltip shows the element's tag, class, and CSS selector.
  4. Click the element you want to fix. The Selector Builder overlay appears.
  5. Use the ancestor dropdown to choose how broad the selector should be (the element itself, its parent, grandparent, etc.).
  6. Optionally check/uncheck class filters and attribute filters to make the selector more specific.
  7. The live preview shows the CSS selector and highlights all currently matched elements on the page.
  8. Click Save Selector. The selector is saved and RTL correction applies immediately.
New selectors default to Force RTL mode — RTL is always applied regardless of content detection. This is the safest default for user-selected elements.

Force RTL vs Auto-detect

Each selector operates in one of two modes:

Auto-detect — The extension analyses the text inside each element. RTL is applied only if the text is detected as Hebrew (first strong character is Hebrew, or ≥ 30% Hebrew letters). Mixed or purely LTR content is left unchanged.

Force RTL — RTL direction is always applied to all matched elements, regardless of content. Use this when auto-detection misses elements or when you always want RTL for a specific section.

Backing up and restoring your configuration

Open the popup, click Actions ▾, then choose Export Config. A JSON file named hebrew-rtl-config-YYYY-MM-DD.json is downloaded containing all your domain configurations.

Your settings also sync automatically across devices via Chrome's built-in sync storage — any device signed into the same Chrome profile will have the same configuration.

Import from file is not yet available. To restore a backup, you will need to re-add selectors manually using the picker.

Managing selectors

Delete a selector — Click the red × button on the right side of any selector row.

Delete all selectors for a site — Open Actions ▾ → Delete All Selectors, then confirm. This clears the entire configuration for the current domain only.

Hover to highlight — Hover over any selector row in the popup to see all elements it currently matches, outlined in blue on the page.

Troubleshooting

Problem What to check
RTL correction is not being applied Check that the master toggle is on and that at least one selector row is enabled (checkbox checked).
Hebrew text is still showing LTR The selector may not be matching the right element. Use + Add Selector to pick the exact element, or hover a selector row to see what it currently targets.
RTL applied to non-Hebrew text The selector is in Force RTL mode. Disable that selector row, or re-add the selector with a narrower CSS path using the picker's ancestor dropdown.
Content loads after the page and misses RTL The extension watches for DOM changes automatically. If content still misses, try reloading the page. (Configurable load delay coming in a future update.)
Popup shows "Not available on this page" The extension only runs on http:// and https:// pages. It cannot run on Chrome system pages (chrome://), the New Tab page, or local files.
Keyboard shortcut does not work Another extension may have claimed the same shortcut. Go to Actions ▾ → Keyboard Shortcuts to reassign it.
